1. Wariant Przeglądarkowy (Najbardziej polecany na hackathon)
Aplikacja webowa to "król" hackathonów – mentorzy i sędziowie mogą ją uruchomić natychmiast pod wygenerowanym linkiem, bez instalowania czegokolwiek.
    Język: JavaScript / TypeScript
        Dlaczego: Absolutny standard w webówce. Pozwala na błyskawiczne postawienie zarówno frontendu, jak i backendu w jednym języku.
        Frameworki: Next.js (React) lub Nuxt.js (Vue). Pozwalają one na tworzenie tzw. Serverless Functions (API Routes). Dzięki temu w jednym projekcie piszesz ładny interfejs, a "pod spodem" bezpiecznie (bez ujawniania kluczy API w przeglądarce) odpytujesz bazy KRS, CEIDG czy modele językowe (OpenAI/Anthropic).
    Alternatywa: Python + Streamlit / FastHTML
        Dlaczego: Jeśli Twój zespół specjalizuje się w Data Science i AI, Streamlit pozwala na napisanie kompletnej aplikacji webowej z gotowymi tabelami, wykresami i polami tekstowymi w ... samym Pythonie, bez dotykania HTML/CSS. Idealne, by kliknąć "Generuj raport ryzyka" i od razu wyświetlić wynik.
2. Wariant Mobilny
Aplikacja mobilna świetnie sprawdzi się jako "narzędzie terenowe" (np. dla spedytora, który na parkingu weryfikuje kierowcę i szybko skanuje telefonem dokumenty).
    Język: Dart (Framework: Flutter)
        Dlaczego: Flutter pozwala na ekspresowe budowanie pięknych, gotowych interfejsów za pomocą gotowych komponentów (Material Design / Cupertino). Na hackathonie nie ma czasu na pisanie osobno na Androida i iOS – Flutter robi to z jednego kodu. Posiada świetne biblioteki do obsługi zapytań HTTP (dio lub http) i natywne wsparcie dla JSON-a.
    Język: JavaScript / TypeScript (Framework: React Native + Expo)
        Dlaczego: Jeśli potrafisz pisać w Reacie, zintegrowane środowisko Expo pozwala na uruchomienie aplikacji na telefonie sędziego w kilkanaście sekund poprzez zeskanowanie kodu QR.
3. Wariant Desktopowy
Aplikacja okienna (Windows/macOS) dla analityka siedzącego w biurze portowym, przetwarzającego setki zgłoszeń.
    Język: C# (.NET / WPF lub .NET MAUI)
        Dlaczego: Jeśli środowiskiem docelowym jest system Windows (bardzo częste w logistyce i portach), C# oferuje potężne, typowane wsparcie dla operacji sieciowych i asynchronicznych (HttpClient, System.Text.Json). Interfejs wyklikasz bardzo szybko, a integracja z zewnętrznymi bibliotekami enterprise jest bezproblemowa.
    Język: JavaScript / Python (Framework: Electron lub Tauri)
        Dlaczego: Budowanie klasycznego desktopu od zera bywa toporne. Tauri (używające Rusta/Webview) lub Electron pozwalają "opakować" aplikację przeglądarkową w plik .exe lub .app. Kodujesz jak stronę internetową, a użytkownik dostaje klasyczny program okienkowy.
